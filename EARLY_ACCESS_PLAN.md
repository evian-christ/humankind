# Early Access Launch Plan

본 문서는 스팀 얼리액세스 출시까지 필요한 작업을 단계별로 정리한 실행 계획입니다. 각 단계는 최소 기능 확보를 목표로 하며, 완료 시 체크 가능합니다.

## 0. 현재 상태 요약
- 위시리스트: 500+ (초기 커뮤니티 존재)
- 스토어 페이지: 등록 완료, 자산 일부 제작됨
- 핵심 루프: 메인 슬롯 배치 → 전투 → 효과 처리 → 선택까지 구현
- 문서 구조: README / SYSTEM_OVERVIEW / DESIGN_REFERENCE 정리 완료

## 1. 기능 스코프 확정 (Week 1)
| 작업 | 설명 | 담당/상태 |
| --- | --- | --- |
| 기능 리스트 잠금 | 얼리액세스 버전에 포함될 시스템 확정 (아래 섹션 기준) |  |
| QA 체크리스트 초안 | 테스트 시나리오/치명적 버그 기준 정의 |  |
| 문서 업데이트 | SYSTEM_OVERVIEW, DESIGN_REFERENCE, README에 스코프 반영 | ✅ (초기 버전 완료) |

## 2. 필수 기능 구현 (Week 1-3)
| 기능 | 세부 내용 | 상태 |
| --- | --- | --- |
| 세이브/로드 | 최소 1 슬롯, 턴/심볼/유물/서브 슬롯 상태 저장 |  |
| 서브 슬롯 UI/로직 | 1×3 슬롯 렌더링, 카운터 표시, 이벤트 트리거 (야만인/자연재해/황금기 등 2~3종) |  |
| 유물 시스템 | 레벨업 → 심볼 선택 → 유물 선택 루프, 유물 5~8개, 희귀도 2단계 |  |
| 골드 사용처 | 최소 1개 (예: 심볼 재롤 비용, 상점 슬롯) |  |
| 승리/패배 연출 | 전용 UI/메시지, 재시작/메인 메뉴 옵션 |  |
| 튜토리얼/헬프 | 기본 흐름 설명 팝업 또는 튜토리얼 시퀀스 |  |

## 3. 품질 보증 & 콘텐츠 다듬기 (Week 3-4)
| 작업 | 설명 | 상태 |
| --- | --- | --- |
| 내부 QA 라운드 | 체크리스트 기반 수동 테스트, 버그 레포트 작성 |  |
| 밸런스 패스 | Food/Enemy/환경 이벤트/유물 효과 조정 |  |
| 사운드/아트 최소화 | 필수 SFX/BGM, UI 폴리싱 |  |
| 세이브 안정성 테스트 | 세이브/로드 반복, 데이터 손상 여부 확인 |  |

## 4. 커뮤니티 & 마케팅 (병행)
| 작업 | 설명 | 상태 |
| --- | --- | --- |
| Devlog/소셜 업데이트 | 진행 상황 주 1회 공유 (GIF, 스크린샷) |  |
| 플레이테스트 모집 | 커뮤니티/디스코드/Reddit 등으로 피드백 수집 |  |
| 뉴스레터/이메일 준비 | 위시리스트 대상 알림 메시지 작성 |  |
| 출시 날짜 확정 | QA 결과 기반으로 날짜 발표 |  |

## 5. 출시 주간 체크리스트
1. 빌드 업로드 및 스팀 리뷰 통과
2. 패치 노트/로드맵 게시 (서브 슬롯 확장, 유물 추가 등 향후 계획 명시)
3. 첫 주 핫픽스 대비 버그 대응 플로우 마련
4. 커뮤니티 채널 모니터링 및 Q&A

## 6. 출시 후 1개월 로드맵
- Week 1: 긴급 버그 픽스, QoL 요청 반영
- Week 2: 신규 유물/서브 이벤트 1세트 추가
- Week 3: 골드 컨텐츠 확장(상점/재롤 옵션 강화)
- Week 4: 통계/밸런스 분석, 다음 대규모 업데이트 계획 발표
