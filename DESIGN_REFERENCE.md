# HUMANKIND - 게임 디자인 레퍼런스 및 방향성 (Game Design Reference)

*본 문서는 반복적인 플레이 및 코드 구현 상태를 바탕으로 작성된 최신 게임 디자인 문서입니다.*

---

## 용어 표기 (Terminology)
- 본 프로젝트의 진행 단위 표기는 **“턴”**으로 통일합니다. (과거 문서/표기에서 사용하던 “스핀(Spin)”은 동일 의미로 취급하며 문서에서는 사용하지 않습니다.)

## 1. 개요 (Overview)
- **장르**: 슬롯머신 기반 덱빌딩 생존 전략 (Slot-builder Deckbuilding Strategy)
- **핵심 목표**: 매 주기 부과되는 식량(Food) 요구량을 충족하여 생존하고, 지식(Knowledge)을 모아 시대를 발전시키며, 같은 슬롯 보드 위에서 시대마다 다른 생산 문법을 다루는 것.
- **주요 테마**: 인류 문명의 발전 (고대의 직접 생산/소모 👉 중세의 증폭/전환/재가공 👉 현대의 연쇄/전역 스케일링).

---

## 2. 핵심 자원 및 컨트롤 (Core Resources)
게임 내 3가지 자원은 각기 다른 목적을 가지며 플레이어의 덱 빌딩 방향을 강제합니다.
- **식량 (Food - 초록색 오각형)**: **생존의 핵심 자원.** 매 10턴마다 기하급수적으로 늘어나는 요구량을 납부해야 합니다. 실패 시 즉시 게임 오버.
- **골드 (Gold - 노란색 원형)**: **경제 및 기회 자원.** 매 턴 주어지는 심볼 선택창에서 더 좋은 기회를 얻기 위해 리롤(Reroll)을 하거나, 좌측 하단의 '유물 상점(Relic Shop)'에서 영구적인 버프를 주는 유물을 구매하는 데 사용됩니다.
- **지식 (Knowledge - 파란색 빛)**: **성장 자원.** 지식 게이지가 찰 때마다 레벨업을 하며, 강력한 팩션/메커니즘 변화를 가져오는 '지식 업그레이드(Knowledge Upgrade)' 카드를 선택할 수 있습니다. 일정 레벨에 도달하면 다음 '시대(Era)'로 넘어가 새로운 심볼 풀이 해방됩니다.

---

## 3. 게임플레이 코어 루프 (Gameplay Core Loop)
1. **턴 (Turn)**: 턴 버튼을 누르면 보드 상의 심볼들이 효과를 발동하며 식량/골드/지식을 생산합니다.
2. **선택 (Selection)**: 제시된 3개의 심볼 중 내 덱의 시너지에 맞는 1개를 선택해 보드의 빈칸에 추가합니다. (건너뛰기 혹은 골드 지불로 리롤 가능).
3. **상점 및 보상 (Rewards & Shop)**: 지식이 모여 레벨업 시 업그레이드 선택창이 나타납니다. 언제든 유물 상점을 열어 골드로 희귀한 유물을 구매할 수 있습니다. (상점 품목은 10턴마다 갱신)
4. **생존 확인 (Survival Test)**: 화면 상단 중앙에 표시된 "백성이 X턴 후 N 식량을 요구합니다"의 기한이 도래하면 식량을 차감합니다. 살아남으면 더 가혹한 다음 주기가 갱신됩니다.

### 3-0. 플레이어 의사결정과 랜덤 배치
- 플레이어가 직접 통제하는 것은 심볼 선택, 리롤/스킵, 지식 업그레이드 선택, 유물 구매 같은 **덱 구성과 선택지 관리**다.
- 보드 위 실제 위치는 매 턴 플레이어 보유 심볼이 무작위로 배치된 결과이며, 플레이어는 특정 심볼을 특정 칸에 놓거나 원하는 인접 구조를 직접 만들 수 없다.
- 따라서 인접, 행/열, 코너, 빈칸 조건은 "배치 퍼즐"이 아니라, 플레이어가 선택한 심볼 구성이 랜덤 배치에서 얼마나 자주 유리한 상황을 만드는지에 대한 확률적 시너지다.
- 설계 문서와 효과 설명에서는 "플레이어가 배치한다", "주변을 채운다", "특정 위치에 놓는다"처럼 직접 배치 조작을 암시하는 표현을 피한다.

### 3-1. 시대가 바꾸지 않는 것 / 바꾸는 것
- **바꾸지 않는 것**: 슬롯 보드, 심볼 선택, 식량/골드/지식 생산, 10턴 주기 식량 납부, 지식 업그레이드, 시대 전환.
- **바꾸는 것**: 플레이어가 같은 자원을 어떤 방식으로 생산하고 증폭하고 전환하는지에 대한 **생산 문법**.
- 시대 차이는 새로운 미니게임을 덧붙여 만드는 것이 아니라, 같은 코어 루프 안에서 심볼 효과의 성격을 달리해 만드는 것이다.

## 4. 메인 메커니즘 및 덱 빌딩 시너지 (Mechanics & Synergies)
단순히 수치가 높은 심볼을 모으는 것을 넘어, 보드 위에서의 상호작용이 핵심입니다.

- **지형 중심 덱빌딩 (Terrain-Centric Deckbuilding)**:
  - 이 게임에서 지형 심볼은 단순 보조 파츠가 아니라, 문명 게임에서의 "영토"에 가까운 핵심 기반입니다.
  - 플레이어는 고대 시대에 확보한 지형을 바탕으로 이후 시대의 운영 방향을 결정하며, 시대가 진행될수록 지형 심볼은 희소해져 기존 기반을 더 값지게 활용해야 합니다.
  - 메인 덱은 자원 종류가 아니라 **지형 기반 운영 방식**으로 구분됩니다. 현재 기준 메인 덱 후보는 `초원`, `평원`, `바다`, `숲`, `열대우림`, `사막`이며, `산`은 특수 포지션으로 취급합니다.
- **공통 생존층 + 열린 빌드 (Shared Survival Layer + Open Build)**:
  - 식량, 지식, 골드, 최소 방어는 특정 덱만의 전유물이 아니라 모든 플레이가 반드시 해결해야 하는 공통 인프라입니다.
  - 따라서 플레이는 하나의 직업형 덱을 고르는 방식이 아니라, 여러 지형 기반 덱을 혼합하면서 각 자원을 자신만의 방식으로 수급하는 열린 빌드를 지향합니다.
  - 종교, 유물, 지식 업그레이드, 전투/방어는 독립적인 메인 덱이라기보다 이러한 지형 기반 덱들을 보정하고 증폭하는 상위 레이어입니다.
- **위치 기반 인접 시너지 (Adjacency Bonus)**: 
  - 특정 심볼은 인접한 위치(주변 8칸)의 다른 심볼에 큰 버프를 줍니다.
  - 단, 이 인접은 플레이어가 직접 배치해서 만드는 퍼즐이 아니라 매 턴 무작위 배치에서 발생하는 확률적 결과입니다.
  - *예시*: `초원`은 `밀`과 `쌀`의 카운터를 더 빠르게 올리고, `평원`은 `소`와 `말`의 식량 생산을 강화합니다. `바다`는 `물고기`, `게`, `진주`의 핵심 기반이 됩니다.
- **자기 파괴와 변이 (Destroy & Mutation)**: 
  - 영구히 자리를 차지하는 심볼 외에도, 일정 턴 이후 효과를 내고 스스로 사라지는 일회성/조건부 심볼들이 보드의 흐름을 원활하게 합니다.
  - *예시*: `바나나` (6턴 후 파괴), `토기` (누적 후 파괴 시 방출), `유물 카라반` (파괴 시 유물 상점 새로고침).
- **시대별 생산 문법 (Era-specific Production Grammar)**:
  - 각 시대는 같은 식량/골드/지식 자원을 다루되, **어떤 방식으로 값을 만들어내는가**가 달라져야 한다.
  - **고대**는 직접 생산, 카운터 누적, 파괴 시 보상, 단순 인접 시너지의 비중이 높다.
  - **중세**는 기존 생산을 증폭하거나 다른 자원으로 전환하고, 랜덤 배치에서 발생한 인접/행/열/전역 조건을 더 크게 해석하는 비중이 높다.
  - **현대**는 여러 심볼의 결과를 연쇄시키고, 보드 전역 상태를 참조해 생산을 크게 키우는 비중이 높다.
- **지식 업그레이드 (Knowledge Upgrades)**: 
  - 보드 전체의 법칙을 바꾸는 강력한 시스템. 단순한 수치 뻥튀기가 아닌 덱의 구조를 바꿀 수 있습니다 (예: 특정 심볼의 추가 소환, 종교 시스템 해방 등).
- **유물 (Relics)**: 
  - 박물관 스탠드에 전시되는 유물들은 보드 밖에 존재하며 영구적인 패시브 능력을 발휘. 이를 기반으로 특정 덱(예: 빙고 조합 덱, 파괴 중심 덱 등)을 특화할 수 있습니다.

## 5. UI / UX 및 비주얼 디렉션 (UI & Visual Direction)
- **프리미엄 픽셀 아트**: 깔끔한 도트 그래픽을 기반으로, 칙칙하지 않고 고급스러우며 동적인 연출 추구.
- **몰입도를 높이는 박물관 감성**: 
  - 특히 유물 상점(Relic Shop) 화면은 위에서 내려오는 극장/박물관 조명광(Spotlight)과 유리 전시장 연출을 통해 프리미엄한 감각을 전달함. 전체적으로 다크/무게감이 있는 배경 테마와 선명한 포인트 색상 사용.
- **쾌적한 애니메이션 피드백**: 
  - 턴 후 자원이 날아가는 연출, 레벨업 시 화면의 전환 처리 등이 매끄러우며, 짧은 호흡의 아케이드적 느낌을 극대화.
- **데이터 브라우저**: 
  - 플레이어가 언제든 인게임 단축키(F3)를 통해 심볼과 유물 정보, 다음 시대의 해금 목록 등을 확인할 수 있는 위키 기능 제공.

### 5-4. 기본 UI 구현 원칙 (미니멀 우선)
- 새로운 오버레이/모달/패널은 사용자가 “스타일을 화려하게” 요청하기 전까지 **초미니멀** 형태로 먼저 구현한다.
- 초미니멀 기본 레이아웃 규격:
  - 배경: `rgba(0,0,0,0.6~0.8)` 같은 단색 반투명
  - 본문: 단순 `border` 또는 얇은 구획선(둥근 모서리/그라데이션/강한 그림자 최소화)
  - 타이포: 통일된 폰트 + 과도한 텍스트 효과(획/그림자/입체효과) 지양
  - 동작: 클릭 배경 닫기 + `Esc` 닫기(가능하면) 같은 기본 UX만 포함
- 이후 단계에서만 요청/가이드가 있을 때:
  - 박물관 감성(조명/포인트 컬러/디테일 프레임), 프리미엄 그림자/그라데이션, 애니메이션을 점진적으로 추가한다.

## 6. 향후 확장 및 구현 목표 (Future Roadmap)

### 6-1. 시대 확장 (Era Expansion)
- (메모) 구현 범위는 반복 개발로 계속 변동될 수 있음.
- 각 시대는 새로운 코어 시스템을 추가하기보다, 같은 코어 루프 안에서 **생산 문법의 무게중심을 이동시키는 것**을 목표로 함:
  - **고대 (직접 생산 / 소모 / 카운터 축적)**:
    - 지형 인접, 일정 턴 누적, 파괴 시 보상 같은 즉물적인 생산 방식이 중심이다.
    - 플레이어는 어떤 지형과 타이머 심볼을 덱에 확보할지에 집중한다.
  - **중세 (증폭 / 전환 / 재가공)**:
    - 이미 깔린 지형과 기존 생산 엔진을 바탕으로, 생산량을 키우거나 다른 자원으로 바꾸거나 더 구조적으로 재해석하는 시대다.
    - 핵심은 새 자원을 만드는 것이 아니라, `직접 생산` 비중을 줄이고 `증폭`, `허브`, `중계`, `재활용`, `자원 전환` 비중을 높여 플레이 감각을 바꾸는 것이다.
    - 중세 심볼은 가능하면 단순 고수치 생산보다, 기존 심볼의 결과를 건드리는 효과를 우선한다.
  - **현대 (연쇄 / 전역 스케일링 / 자동화 감각)**:
    - 중세의 증폭과 전환 위에서, 여러 심볼의 결과가 이어지거나 보드 전체 상태를 참조하는 대규모 생산이 중심이 된다.
    - 현대는 중세보다 더 넓은 범위와 더 큰 규모의 계산을 담당하며, 시대 후반의 폭발력을 맡는다.

### 6-2. 전투 및 군사 시스템 연동 (Combat System)
- 적 심볼(야만인, 침략자) 등장 매커니즘 정교화.
- 방어력(HP)과 공격력(ATK) 스탯을 활용한 전투 페이즈 도입 예정.
  - 지식 및 골드 성장에만 치중한 '순수 경제 덱'이 외부의 물리적 침략에 무너지지 않도록 전투 심볼 배치가 강제되는 밸런싱.

### 6-3. 다양성 (Diversity)
- 심볼/유물/업그레이드 후보는 계속 추가/삭제/리밸런싱되는 전제.
- 장기적으로 심볼 풀을 크게 확장해, 로그라이크 덱빌더 특유의 매 판 다른 조합(Replayability)의 재미를 증대.
- 단, 덱 구조는 폐쇄적인 직업형 덱보다 여러 지형 기반 엔진을 창의적으로 섞어 쓰는 방향을 우선한다.
- 특히 시대별 신규 심볼은 "새 시스템 추가"보다 "기존 생산 문법을 어떻게 바꾸는가"를 기준으로 평가한다.
