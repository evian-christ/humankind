# HUMANKIND - 덱 구조 및 시대 확장 메모

## 목적
- 본 문서는 현재 게임의 덱 구조를 어떤 관점으로 해석해야 하는지, 그리고 그 구조를 바탕으로 중세를 어떻게 설계할지 정리하기 위한 작업 문서입니다.
- 특히 "자원축"과 "메인 덱"을 혼동하지 않기 위한 기준을 명문화합니다.

## 1. 덱을 보는 기본 관점
- 이 게임에서 **식량 / 지식 / 골드 / 최소 방어**는 특정 덱만의 특징이 아니라 모든 플레이가 반드시 해결해야 하는 공통 생존 인프라입니다.
- 따라서 `농업 식량 덱`, `지식 덱`, `골드 덱`, `종교 덱`처럼 자원 종류만으로 메인 덱을 구분하지 않습니다.
- 메인 덱은 "어떤 지형 기반을 중심으로 생존을 풀어내는가"로 분류합니다.
- 종교, 유물, 지식 업그레이드, 방어/전투는 독립 메인 덱이 아니라 지형 기반 덱 위에 얹히는 증폭층 또는 보정층으로 다룹니다.

## 2. 지형 중심 설계 원칙
- 지형 심볼은 단순한 시너지 파츠가 아니라, 문명 게임에서의 "영토"와 유사한 핵심 기반입니다.
- 고대 시대에는 지형 심볼이 일반 심볼과 비교적 비슷한 비중으로 등장해, 플레이어가 문명의 기반을 구축하는 느낌을 줍니다.
- 시대가 진행될수록 지형 심볼은 점차 얻기 어려워지며, 플레이어는 새 지형을 계속 확보하기보다 이미 가진 지형을 더 효율적으로 활용해야 합니다.
- 이 희소성은 "고대에 어떤 땅을 확보했는가"가 이후 시대의 정체성을 좌우하게 만드는 핵심 장치입니다.

## 2-1. 랜덤 배치 전제
- 이 게임에서 플레이어가 직접 통제하는 것은 **어떤 심볼을 덱에 추가할지**, 어떤 지식 업그레이드를 고를지, 리롤/스킵/상점 사용으로 선택지를 조정할지이다.
- 보드 위 실제 위치는 매 턴 플레이어 보유 심볼이 **무작위로 배치된 결과**이며, 플레이어가 특정 칸에 심볼을 놓거나 인접 구조를 직접 설계하지 않는다.
- 따라서 덱/업그레이드 설계에서 "어디에 배치할지", "특정 심볼 주변을 어떻게 채울지" 같은 표현은 플레이어 의사결정으로 취급하지 않는다.
- 인접, 같은 행/열, 빈칸, 코너 같은 위치 조건은 플레이어가 직접 만든 퍼즐이 아니라, 덱에 넣은 심볼 구성이 무작위 배치에서 얼마나 자주 좋은 상황을 만드는지에 대한 **확률적 시너지**로 해석한다.
- 좋은 시대 확장은 특정 배치를 요구하기보다, 플레이어가 선택한 심볼 조합이 랜덤 보드 위에서 반복적으로 의미 있는 결과를 만들도록 해야 한다.

## 3. 현재 메인 덱 분류
- 초원덱
- 평원덱
- 바다덱
- 숲덱
- 열대우림덱
- 사막덱

### 보조/특수 포지션
- 산덱은 메인 덱이 아니라 특수 포지션으로 본다.
- 오아시스는 독립 덱이 아니라 사막덱 내부 파츠로 취급한다.

## 4. 열린 빌드 원칙
- 플레이어는 하나의 덱을 선택해서 고집하도록 강제되지 않는다.
- 실제 플레이에서는 여러 지형 기반 덱을 섞어 쓰는 경우가 기본값에 가깝다.
- 따라서 각 덱은 완전히 폐쇄된 파츠 묶음이 아니라, 다른 지형 기반 덱과 자연스럽게 얽히는 브릿지 심볼을 가질 수 있어야 한다.
- 장기적으로는 지금보다 더 각 덱 간 연결 심볼을 늘려, "정해진 정답 빌드"보다 창의적인 혼합 운영을 유도한다.

## 5. 메인 덱과 공통 운영층의 관계

### 공통 운영층
- 식량: 어떤 플레이든 생존을 위해 계속 필요
- 지식: 시대 전환과 성장에 계속 필요
- 골드: 리롤과 유물 확보를 위해 계속 필요
- 최소 방어: 야만인 및 외부 위협 대응을 위해 계속 필요

### 메인 덱층
- 위 공통 운영층을 어떤 지형 중심 구조로 해결하느냐가 덱의 개성이다.
- 예를 들어 초원덱과 바다덱은 모두 식량을 해결하지만, 필요한 지형 구성, 보조 자원 수급 방식, 랜덤 배치에서 기대하는 시너지 빈도, 약점이 서로 다르다.

### 증폭층
- 종교: 한 종류만 강하게 택해 덱의 방향성을 크게 보정하는 강력한 특수 레이어
- 유물: 특정 메인 덱이나 하이브리드 구조를 밀어주는 장기 버프
- 업그레이드: 특정 심볼 해금, 기본 규칙 변경, 전환 촉진
- 전투/방어: 모두에게 요구되지만, 어떤 덱은 이 축을 더 적극적으로 활용할 수 있다

## 6. 중세 설계에 대한 현재 결론
- 중세는 새로운 자원축을 추가하는 시대가 아니다.
- 중세는 **고대에 확보한 지형 기반의 생산 방식을 바꾸는 시대**다.
- 즉, 중세 설계의 핵심 질문은 "어떤 자원을 주는가?"보다 "같은 식량/골드/지식을 어떤 방식으로 생산하게 만들 것인가?"이다.

### 중세에서 유지해야 할 원칙
- 고대의 여섯 메인 덱이 중세에도 이어져야 한다.
- 중세는 새 메인 덱을 발명하기보다 기존 메인 덱을 심화, 전환, 완충하는 방향이어야 한다.
- 종교는 "종교덱"이 아니라 기존 메인 덱들에 강한 방향성을 부여하는 특수 증폭 장치여야 한다.
- 전투/방어는 독립 메인 덱이 아니라 기본 생존층으로 남기되, 일부 덱은 더 공격적으로 활용할 수 있다.
- 중세는 코어 루프를 바꾸지 않는다. 대신 `직접 생산`보다 `증폭`, `전환`, `재가공`, `허브화` 비중을 높여 플레이 감각을 바꾼다.

### 중세에서 기대하는 변화
- 고대: 지형을 확보하고, 인접과 카운터를 통해 직접 생산하는 기초 엔진을 만든다
- 중세: 이미 만든 엔진을 증폭하고, 다른 자원으로 전환하고, 보유 심볼 구성이 랜덤 배치에서 만들어내는 확률적 시너지의 가치를 키운다
- 현대: 여러 생산 결과를 연쇄시키고, 보드 전역 상태를 참조해 스케일을 키운다
- 결과적으로 같은 초원덱, 같은 바다덱이라도 시대에 따라 "무엇을 더 많이 생산하느냐"보다 "어떻게 생산하느냐"가 달라져야 한다

## 7. 이후 브레인스토밍 순서
1. 여섯 메인 덱 각각의 고대 핵심 감각을 정의한다.
2. 각 메인 덱이 현재 어떤 약점과 병목을 가지는지 정리한다.
3. 각 메인 덱이 중세에서 `증폭 / 전환 / 재가공 / 허브화` 중 무엇을 받는지 정한다.
4. 각 메인 덱이 현대에서 `연쇄 / 전역 스케일링` 중 무엇으로 확장되는지 정한다.
5. 그 다음에야 시대별 심볼, 업그레이드, 브릿지 심볼을 설계한다.

## 8. 현재 개발 기준 시사점
- 이제 목표는 데모용 중간 지점이 아니라, **중세를 하나의 완성된 시대 문법으로 확립하는 것**이다.
- 따라서 중세의 성공 여부는 새 심볼 수보다도, 플레이어가 고대에 만든 기반이 중세에서 `직접 생산 중심 엔진`에서 `증폭/전환 중심 엔진`으로 얼마나 명확하게 넘어가는지에 달려 있다.
- 이 축이 선명해야 다음 시대인 현대도 중세의 상위 호환이 아니라, `연쇄/전역 스케일링`이라는 별도 생산 문법으로 자연스럽게 올라갈 수 있다.

## 9. 초원축 농업 업그레이드 방향
- 초원축은 안정적으로 식량을 생산하는 축으로 유지한다.
- 약점은 골드, 지식, 전투 확장성이 낮다는 점이며, 이 약점은 의도적으로 유지한다.
- 중세 이후 초원축은 신규 심볼을 추가하지 않고, 이미 존재하는 `초원`, `밀`, `쌀`의 효과 변경과 강화만으로 발전한다.
- 이 축은 안정적인 만큼 플레이 체감이 다소 보수적이고 stale하게 느껴져도 괜찮다. 새 파츠를 소개하는 대신, 고대에 확보한 농업 기반이 더 효율적인 식량 엔진으로 굳어지는 방향이다.
- 효과 설계는 특정 배치를 요구하지 않는다. `인접한 초원`은 매 턴 무작위 배치에서 발생한 보너스이며, 이후 단계의 `보드 위 초원 수` 참조는 인접 운 의존도를 낮추고 초원 보유량 자체의 가치를 키우기 위한 장치다.

### 레벨 17: 농업 잉여
- 의존성: `삼포제`
- 신규 심볼: 없음
- 대상 심볼: `밀`, `쌀`
- 효과:
  - 기존 `밀`의 "인접한 초원마다 +1 카운터" 효과를 "인접한 초원마다 +2 카운터"로 강화한다.
  - `쌀`도 동일하게 "인접한 초원마다 +1 카운터"에서 "인접한 초원마다 +2 카운터"로 강화한다.
- 의도:
  - 고대의 인접 기반 카운터 문법을 그대로 이어받되, 중세 초원축의 식량 회전율을 높인다.
  - 결과물은 식량으로만 유지해 초원축의 안정성과 약점을 동시에 보존한다.

### 레벨 23: 현대 농업
- 의존성: `농업 잉여`
- 신규 심볼: 없음
- 대상 심볼: `밀`, `쌀`
- 효과:
  - `밀`은 더 이상 "인접한 초원마다 +2 카운터"를 사용하지 않고, "보드 위 초원 1개당 +1 카운터"를 얻는다.
  - `쌀`도 동일하게 "보드 위 초원 1개당 +1 카운터"를 얻는다.
- 의도:
  - 농업축이 인접 운에 덜 묶이고, 덱에 확보한 초원 수 자체를 직접적인 생산 안정성으로 전환한다.
  - 고대와 중세의 인접 문법을 완전히 버리는 것이 아니라, 후속 시대에서 보유 기반 전체를 참조하는 방식으로 확장한다.
